# v1.2.2 Outstanding Issues

Captured after Notion raw API integration (v1.2.1 scope). Pre-existing issues only — none introduced by the Notion refactor.

## App.vue

- **`isDesktop` not reactive to window resize** (line 374)
  - `const isDesktop = window.innerWidth > 768` is computed once at module load
  - Sidebar initial state won't adapt if user resizes or rotates device after load
  - Fix: use a reactive media query (`window.matchMedia`) or `ResizeObserver`

- **File download race condition** (lines 230–247)
  - `pendingDownloadPath` is a single `ref` — if two downloads are triggered before the first `files:read:result` arrives, only the last one fires
  - Fix: use a `Map<path, resolve>` or queue instead of a single ref

## SettingsModal.vue

- **No token validation before save** (Slack tab)
  - User can clear the bot token field and save, overwriting a valid stored token with empty string
  - Fix: warn or block save if `enabled` is true but token is empty

- **Analyse Structure silently overwrites manual field mapping edits** (Notion tab)
  - Clicking "Analyse Structure" auto-populates `fieldMappings` without confirmation
  - If user has manually curated mappings, they are lost without warning
  - Fix: show a confirm dialog, or only populate if mappings are currently empty

- **`isUpdatingFromProps` flag uses `setTimeout(0)` to prevent watch loop** (line 76–81)
  - Fragile timing assumption; could fail under rapid successive server updates
  - Fix: use a proper dirty-check or Vue's `nextTick` with a more deterministic guard

## server/slack/config.js

- **Silent failure on malformed `~/.claude.json`** (line 71)
  - MCP token fallback swallows parse errors silently; no log emitted
  - Low severity — MCP config is a best-effort fallback, but hard to debug if misconfigured
  - Fix: add a `logger.warn` on parse error

## server/slack/lib/classifier.js

- **`cachedClaudeMd` never invalidated at runtime**
  - CLAUDE.md is read once and cached for the server lifetime
  - `clearClaudeMdCache()` is exported but never called internally; requires server restart to pick up edits
  - Low severity for most usage, but could surprise during active CLAUDE.md editing sessions
  - Fix: call `clearClaudeMdCache()` on Slack config save, or add a TTL (e.g. 10 min)
